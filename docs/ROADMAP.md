# カツアゲくん v2 — ロードマップ

> 更新: 2026-03-10 06:40 JST

## 🔴 大原則: 必ず動く優先

**v2の最優先目標は「テキスト回答を100%確実に拾う」こと。**
複雑な機能は後回し。シンプルに、確実に、壊れない。

---

## Phase A: 確実に動く最小版（今ここ）
- テキスト回答のみ対応
- busy→not busy 遷移検知（MutationObserver禁止）
- Canvas/ダウンロードファイル/sandbox → 完全無視
- 目標: **10回連続テスト 100%成功**

## Phase B: ファイル出力の検知だけ追加
- GPT回答後にダウンロードファイルが残ってるかを検知
- 「ファイルあり」フラグだけreceiverに送る
- **ダウンロード自体はしない** → ロブ🦞が手動で取りに行く
- 検知方法候補:
  - ダウンロードボタンの存在チェック（DOM）
  - Canvas出力パネルの有無チェック
- テキスト回答の取得に影響を与えないこと（Phase A最優先）

## Phase C: ファイル自動ダウンロード（必要になったら）
- Phase Bで「手動DLが面倒すぎる」と判明した場合のみ実装
- Tampermonkey拡張でDLリンクをクリック→保存
- **Phase Aが100%安定してる前提。不安定化するなら入れない**

---

## 設計判断の記録

| 決定 | 理由 |
|------|------|
| v1のMutationObserverを廃止 | 完了検知が不安定だった元凶 |
| Canvas/ファイル対応を後回し | v1の複雑化原因。まず動くものを |
| 行数制限撤廃 | 本質じゃない。禁止事項を守れば十分 |
| ファイル検知はフラグだけ | DLの自動化は壊れやすい。必要になるまでやらない |
| 100%動くのが最優先 | やまちゃん🗻指示（2026-03-10） |
